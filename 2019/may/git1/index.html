Git命令

写一下git的命令以便忘记时查找

获取帮助

git help <verb>
git <verb> --help
man git-<verv>

config

  • git config针对一个git仓库
  • git config –global针对一个用户
  • sudo git config –system针对一个系统，因为是针对整个系统的，所以必须使用sudo

个人信息

用于提交git时显示的个人信息

git config --global user.name "your name"
git config --global user.email "your email"

仓库独立指定

git config user.name "your name"
git config user.email "your email"

不加 - -global

配置编辑器

默认缺省编辑器 为vi or vim。 如想使用emacs

git config --global core.editor emacs

配置比较工具

用于解决合并时的冲突，如vimdiff

git config --global merge.tool vimdiff

配置别名

如 git status 设置别名 st

git config --global alias.st status

推荐设置别名

git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

输入q退出

检查配置

git config --list

删除config key值

git config --global --unset key

.gitignore文件

设置文件监控规则 文档描述gitignore语法规则有五条：

  • 忽略空白行或以#开始的行（注释）；
  • 按照标准glob工作，并递归应用于整个工作目录（参考glob）；
  • 模式以/打头，禁止递归工作；
  • 模式以/结尾，限定特定目录；
  • 使用感叹号!对模式含义取反；

gitignore规则细化补充如下：

  • 通用的规则应该写在前面，专有的放后面。
  • 规则中不含有/的，则当作通配符处理，每个目录迭代检查；
  • 规则以/结尾的，仅限于目录，而不包含同名的链接和文件；
  • 模式以连续两个星号和/打头(**/)时，和移除**/含义相同；
  • 模式以连续两个型号结尾时，表示匹配其下一切内容，含子目录；

克隆git仓库到本地

clone

git clone url //支持git、http(s)、ssh等协议

默认只有master分支

链接分支
  • 在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致
  • 建立本地分支和远程分支的关联，使用git branch –set-upstream branch-name origin/branch-name

fetch

拉取远程更新到本地，可指定分支 对本地代码无影响

git fetch <shortname> <分支名>

pull

拉取远程分支的更新并合并 远程分支与当前分支合并可省略:后部分

git pull <shortname> <远程分之名>:<本地分支名>

默认在本地分支与远程分支建立追踪关系 也可以手动建立追踪关系

git branch --set-upstream <分支> \<分支>

如果远程分支已删除 pull不会删除本地分支

git pull -p //本地删除远程已删除的分支

push

推送本地更新，与 pull相仿 如果远程分支不存在，则新建

git push <shortname> <本地分支>:<远程分支>

省略本地分支名 等于删除远程分支

git push <shortname> :<远程分支>

等同

git push <shortname> --delete master

如果存在多个远程追踪关系，使用-u指定一个默认主机

添加文件

git add .  // 把所有变化提交到暂存区 包括修改和新建 不包括删除
git add -u // 把添加的文件中修改的文件提交到暂存区(包含删除) 不添加新文件
git add -A // all
git add <file>... // 可添加多个文件

官网文档

查看当前文件状态

git status

查看已暂存和未暂存的更改

git diff

以暂存的和上次提交的更改

git diff --cached 
git diff --staged

提交

git commit //启动默认编辑器输入更新说明
git commit -m "更新内容" // 加-m 在之后输入更新说明

跳过使用暂存区域

git commit -a // 跳过 add 步骤

移除文件

  • rm <file>... 移除版本控制并从工作目录中删除
  • git rm -f <file>... 移除已添加到暂存区的文件
  • git rm --cached <file> 不进行版本控制但保留文件

移动文件 or 改名

git mv file_from file_to

运行git mv相当于下面三条命令

mv README.txt README
git rm README.txt
git add README

撤销修改

使文件回到最后一次add or commit 时的状态

git checkout -- <file>

版本回退

git log  查看记录
  • HEAD 表示当前版本
  • 上一个为 HEAD^
  • 上上个为 HEAD^^
  • 上100个为 HEAD~100

也可指定回退版本的commitid - -hard 彻底回退到某个版本 - -soft 只回退 commit 信息

git rest --hard HEAD^

撤销回退

git rest --hard commitid

git reflog 记录HEAD的修改

远程库管理

列出管理的远程库

git remote -v

本地关联远程库

git remote add <shortname> url

远程库重命名

git remote rename <oldname> <newname>

删除关联远程库

git remote rm <shortname>
